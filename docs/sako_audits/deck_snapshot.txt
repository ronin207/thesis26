===== SLIDE 5 (Problem Statement) =====
Problem Statement
Waseda University Sako Laboratory
Page 5
Research Question
BDEC proves its relations with a fixed Aurora circuit. A presentation-structure or signature change forces re-deriving and re-auditing that circuit. In a general-purpose zkVM the same change is a guest-program edit.
The decision belongs to the operator of a credential ledger (a university consortium issuing digital diplomas): can holders generate the privacy-protecting proof on their own machines, and what does each future scheme or parameter change cost in regenerated and re-audited artefacts?
This thesis asks whether a general-purpose zkVM is a viable substrate for post-quantum anonymous credentials on consumer hardware. BDEC-PLUM is the use case; the comparison of proving substrates is the contribution. We locate where the wall sits and what would move it.

===== SLIDE 8 (The Field-Mismatch Cost) =====
The Field-Mismatch Cost
Waseda University Sako Laboratory
Page 8
To achieve the required flexibility ("late-binding"), the industry is moving toward general-purpose zkVMs, which allow developers to write dynamic policies in high-level software
When replacing static circuits with zkVMs, we must explicitly state they are not equivalent environments for PQC
Classical cryptography currently works in zkVMs only because developers built custom hardware shortcuts called "precompiles". For instance, a classical SHA-256 hash takes only ~68 cycles because it enjoys a precompile. However, post-quantum algebraic hashes required by PLUM's 192-bit prime field (e.g., Griffin) have no precompiles, forcing the zkVM to emulate 192-bit (about 7 BabyBear limbs) math inside a 31-bit architecture. This takes ~2,000 to 3,000 cycles per permutation.

===== SLIDE 17 (Where the Wall Sits) =====
Where the Wall Sits
SP1: the succinct receipt is not zero-knowledge (official security model). The ZK wrap (PLONK) is OOM-killed (out of memory) at ~20 of 24 GB. The only wraps offered are Groth16/PLONK, both pairing-based, hence not post-quantum.
RISC Zero: the succinct receipt is blinded, but the vendor explicitly disclaims a formal zero-knowledge guarantee, so BDEC's anonymity theorems cannot be discharged on it. Its Groth16 wrap is x86-only and does not run on this machine (Apple Silicon).
Conclusion: more memory removes neither obstruction. Both substrates fail to reach a post-quantum zero-knowledge proof, each for its own reason.
This is the wall this thesis locates by measurement: the base proofs of both relations are measured (27-31 / 40-44 min); only the ZK wrap remains.




Flexibility (the third corner) is reached on both substrates (the update-churn re-audit footprint); what stays empty is post-quantum and zero-knowledge together.

===== SLIDE 18 (Open Items) =====
Open Items
T2, Griffin AIR soundness: the per-row layer is proven; the lookup-borne families (round count, memory binding, cross-row threading) still need a full-prover harness.
RISC Zero prove anomaly: localized to the vendor per-segment self-check; our code is exonerated by the composite re-run; reporting upstream.
Two inherited deviations from published Griffin (the non-MDS circulant; L3 reads the updated lane): disclosed in the draft; the cited security analysis must be re-checked against them.
Prime instantiation: the paper p0 is composite, so we use a nearby prime; needs author confirmation.
Wall-clock variance: thermal state alone moves identical runs 1.44x; every headline figure carries its run count and settings at the point of claim.
Griffin cryptanalysis (FreeLunch, CRYPTO 2024 + 2025 successors): the 14-round count is sized against the superseded FGLM model, so its 128-bit margin is unvalidated (not broken); resolve via the authors’ estimator or migration to Poseidon2.
