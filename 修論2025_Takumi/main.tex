    \begin{coverabstract}
       Post-quantum anonymous credentials need a signature whose verification can be proved in zero knowledge. We ask whether the algebraic-hash PLUM signature can be verified inside a general-purpose zero-knowledge virtual machine on a consumer machine (Apple M5~Pro, $24$~GB), against a Loquat-instantiated Aurora static circuit as the reference. Without a custom precompile, a single PLUM verification does not complete on this machine: it exhausts memory under default settings and does not terminate within three hours under memory tuning. We build a Griffin-$\mathbb{F}_p^{192}$ precompile that removes the field-emulation tax. With it, the same verification is proved in $14.89$ to $32.5$ minutes (build- and machine-state-sensitive; Section~\ref{sec:evaluation}) on SP1 at the $80$-bit setting, under a documented reduced-memory configuration. The figure is conditional on the constraint-level soundness of the bespoke chip, whose per-row layer we prove and whose lookup-borne remainder we carry as an open premise; the paper's $128$-bit setting is untested and expected to exceed the memory budget. Feasibility for the standalone signature is therefore reached on this hardware. The binding constraint then moves up the stack. The receipt this proving produces is succinct and carries no zero-knowledge guarantee, the zero-knowledge wrap that anonymity requires exhausts memory before it completes, and the only wraps the toolchain offers are pairing-based and so not post-quantum. These walls are measured at the standalone verification, the cheapest relation in the credential system, and every BDEC relation contains it, so deployable post-quantum anonymous credentials are not reached on this stack. Both BDEC relations are themselves proved in succinct, non-zero-knowledge mode on SP1 (credential generation in about $27$--$31$ minutes, presentation in about $40$--$44$ minutes), so the wall is the zero-knowledge wrap rather than the base proof. The static reference, with zero knowledge enabled, produces a post-quantum zero-knowledge proof of its single-verification building block in about $22$ minutes (mean of three runs, $21.4$--$22.9$) on the same machine. We locate where the wall sits and what would move it. We also measure update churn in both regimes. Against the transparent, non-preprocessing Aurora baseline the recompile cost is sub-second and the advantage survives only as a regeneration-and-re-audit footprint; against a preprocessing baseline (Fractal) the advantage is real, the per-circuit indexing itself exceeding the memory budget at the BDEC size.
    \end{coverabstract}
