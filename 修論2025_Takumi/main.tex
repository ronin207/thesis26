    \begin{coverabstract}
       Post-quantum anonymous credentials require a signature whose verification can be proved in zero knowledge. We ask whether the algebraic-hash PLUM signature can be verified inside a general-purpose zero-knowledge virtual machine on a consumer machine (Apple M5~Pro, $24$~GB), against a Loquat-instantiated Aurora static circuit as the reference. Without a custom precompile, a single PLUM verification does not complete: it exhausts memory or fails to terminate within three hours. We build a Griffin-$\mathbb{F}_p^{192}$ precompile that absorbs the field-emulation tax; with it the verification proves in about $14.25$ minutes (mean of three fixed-config runs) at the $80$-bit level, conditional on the constraint-level soundness of the chip. Feasibility for the standalone signature is reached, but the binding constraint moves up the stack: the receipt produced is succinct yet not zero-knowledge, and the only zero-knowledge wraps the toolchain offers are pairing-based, hence not post-quantum, or are blocked by a structural recursion-shape provisioning wall that more memory does not remove. The Aurora reference, with zero knowledge enabled, produces such a proof in about $22$ minutes. Deployable post-quantum anonymity is therefore not reached on this stack: we locate this dual obstruction, name the levers that would move it, and report update churn as a secondary deployment-lifecycle cost.
    \end{coverabstract}
