    %概要
    \begin{coverabstract}
       Post-quantum anonymous credentials need a signature whose verification can be proved in zero knowledge. We ask whether one such scheme, the algebraic-hash PLUM signature, can be verified inside a general-purpose zero-knowledge virtual machine on a consumer machine, and we report a frontier result with a clear sign. Without a custom precompile, a single PLUM verification exhausts memory on $24$~GB and cannot be proved at all. With a bespoke large-field Griffin-$\mathbb{F}_p^{192}$ precompile that removes the field-emulation tax, the same verification is proved in $32.5$ minutes on SP1 at the $80$-bit setting, the largest that completes within the memory bound; the paper's $128$-bit setting does not terminate, which is itself part of the frontier. Feasibility for the standalone signature is therefore reached, but the binding constraint then moves up the stack. The zero-knowledge wrap that anonymity requires exhausts memory at $24$~GB, and the only wrap that fits is pairing-based and so not post-quantum. Deployable post-quantum anonymous credentials are not reached on this stack, and we locate exactly where the wall sits and what would move it. The proof we run is a succinct receipt and is not itself zero-knowledge. We also report update churn for both regimes; its wall-clock cost is sub-second against a transparent non-preprocessing argument, so the recompile burden does not separate the two substrates and survives only as a re-audit footprint argument conditional on the preprocessing model. We measured each substrate on consumer hardware (Apple M5~Pro, $24$~GB), against a Loquat-instantiated Aurora static-circuit reference.
    \end{coverabstract}
